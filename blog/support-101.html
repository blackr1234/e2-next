1 故障

要求對方提供故障既基本背景：
  1. How urgent do you need this issue to be resolved?
  2. What is the impact of this issue?
  3. Is there any financial impact because of this issue?
  4. Is this issue impacting the end users (or business users)?
  5. In which environments did this issue occur?
  6. Did this issue occur in production environments?
  7. When did this issue occur?
  8. Is this issue recurring? When was the first time it occurred? How often does it occur?
  9. Is this issue intermittent?
  10. When was the last time that it worked?
  11. When was the last time that this issue occurred?
要求對方提供系統資料以及檔案：
  1. Please provide the application logs and the error logs in text files.
  2. Please provide the source code of this application.
  3. Please provide the Maven pom.xml file of this application.
  4. Please provide the configurations (i.e. environment variables, program arguments, config files) of this application.
  5. Please provide the architecture diagram of this application.
  6. Please provide the API flow or program flow, and explain where this issue occurred.
  7. Please provide a minimal, reproducible example that can demonstrate this issue.
引導對方：
  1. Were there recent changes in the application (code / config) that possibly caused this issue?
  2. Have you tried reverting the recent changes in the application (code / config)? Did it help?
  3. Were there recent outages in the involved environments? Have you contacted the cloud platform service provider?
  4. What action was performed when the issue happened?
  5. What are the conditions for this issue to occur?
  6. What have you done to troubleshoot this issue? Are there any useful findings or root cause analysis that you could share with us?
  7. What have you done to attempt to resolve this issue? Is there a workaround?
  8. Is there another application that works in the same environment? Have you compared the implementation (code / config) with the application that works?
  9. Is this a new application? Did the previous version of this application work?
  10. Which component do you suspect to be faulty?
  11. Please explain why you think that this issue is related to our products.

2 新需求

  1. Please elaborate on the background of this feature request.
  2. Which parties are requesting for this feature?
  3. Who are the end users?
  4. What are the use case scenarios?
  5. How urgent do you need this feature?
  6. What is the soft deadline for this feature to be implemented?
  7. Is there a hard deadline for this feature to be implemented?
  8. What is the impact, including financial impact, if this feature is not implemented at all?
  9. What is the impact, including financial impact, if this feature is not implemented on time?
  10. Will it introduce new costs after this feature is implemented?
  11. Will there be any impact (functional / non-functional) to other systems if this feature is implemented?
  12. How do the users do it today?
  13. Is there any workaround or alternative way to achieve this requirement?
  14. Who can help us conduct testing (e2e / UAT) once this feature is implemented?
