2.2 Java code
Producer：
  • 因為 Spring Cloud Stream 每秒會發送一個訊息，作為 demo purpose，我地可以用系統時間既秒數黎做個 Kafka 訊息既 key。
  • 咁 Kafka 就會根據個 key 既 hash 計算，達到平均分配啲訊息去個 topic 既各個 partitions 既效果。
MyProducer.java：
1@Slf4j
2@Configuration
3public class MyProducer {
4
5    @Bean
6    public Supplier<Message<String>> demoProduce() {
7        return () -> {
8            final String dateTime = LocalDateTime.now(ZoneId.of("Hongkong"))
9                                        .format(ofPattern("uuuu-MM-dd HH:mm:ss"));
10            final long key = (System.currentTimeMillis() / 1000) % 10 + 1;
11            final String msg =
12                    format("key(%s) - Test msg - %s (%s)",
13                           key, RandomStringUtils.secure().nextAlphanumeric(8), dateTime);
14
15            log.info("Sending: {}", msg);
16
17            return MessageBuilder.withPayload(msg)
18                    .setHeader(KafkaHeaders.KEY, String.valueOf(key).getBytes())
19                    .build();
20        };
21    }
22}
MyConsumer.java：
1@Slf4j
2@Configuration
3public class MyConsumer {
4
5    @Bean
6    public Consumer<String> demoConsume() {
7        return msg -> {
8            log.info("Received message: {}", msg);
9        };
10    }
11}
DemoController.java：
1@RestController
2public class DemoController {
3
4    @Autowired
5    StreamBridge streamBridge;
6
7    @PostMapping
8    public void send(@RequestParam(name = "msg") String msg) {
9        streamBridge.send("demoProduce-out-0", msg);
10    }
11}

application-consumer.yml：
spring:
  cloud:
    function:
      definition: demoConsume
application-producer.yml：
spring:
  cloud:
    function:
      definition: demoProduce # 注意：demoProduce 會每秒發送訊息

3 測試

3.1 自動創建 topic
用 producer profile 運行呢個 microservice：
java -jar app.jar --server.port=8080 --spring.profiles.active=producer
啟動左個 producer microservice 之後，執行以下 command 查詢 topics：
kafka-topics.sh --bootstrap-server localhost:9092 --describe
可以見到多左以下既野：

3.2 自動發送訊息
Producer microservice 用左 spring.cloud.function.definition 註冊既 Supplier functions 會自動每秒執行，即係每秒都會有一個新訊息發送到 Kafka。

3.3.3 兩個 consumer groups
運行多一個 consumer profile 既 instance，不過今次將 consumer group 改成 demo-group-2：
java -jar app.jar --server.port=8083 --spring.profiles.active=consumer --spring.cloud.stream.bindings.demoConsume-in-0.group=demo-group-2
亦可以運行多一個 instance：
java -jar app.jar --server.port=8084 --spring.profiles.active=consumer --spring.cloud.stream.bindings.demoConsume-in-0.group=demo-group-2
啟動左之後，我地會發現：
  • 當我地第一次用新既 consumer group demo-group-2 黎接收訊息，佢既 consumers 會收到個 topic 由創建到而家既所有訊息，亦即係之前既 consumer group demo-group 既 consumers 接收過既所有訊息。
    • 當 demo-group-2 只有 1 個 consumer，無論個 topic 有幾多個 partitions，呢個 consumer 都會收到個 topic 既所有 partitions 既訊息。
    • 當 demo-group-2 有 2 個 consumers，而個 topic 有 2 個 partitions，咁佢地就可以一對一，每個 consumer 就會收到個 topic 其中一個 partition 既訊息。
  • Consumer groups demo-group 以及 demo-group-2 既 consumers 都會繼續接收到一樣既訊息，而各個 consumer group 裡面既係 competing consumers。
    • 即係每一個訊息都會被發送到所有 consumer groups，但每個 consumer group 裡面只會由其中一個 consumer 接收。
