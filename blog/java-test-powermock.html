4.2 寫 Java code

先建立一個典型既 Spring Boot web application。

寫一個 dummy component：

@Component
public class MyComponent {

}

之後我地可以寫 test code：

@PrepareForTest({ StringUtils.class, RandomStringUtils.class })
@PowerMockIgnore({ "com.sun.org.apache.xerces.*", "javax.xml.*", "org.xml.*", "javax.management.*" })
@SpringBootTest
@RunWith(SpringRunner.class)
public class MockStaticSpringTest {

    @Autowired
    MyComponent myComponent;

    @Rule
    public PowerMockRule powerMockRule = new PowerMockRule();

    @Before
    public void setUp() {
        PowerMockito.spy(StringUtils.class);
        PowerMockito.mockStatic(RandomStringUtils.class);
    }

    // Spring Boot test，可以測試 Spring context 能唔能夠成功 load 到
    @Test
    public void contextLoads() {
        assertNotNull(myComponent);
    }

    @Test
    public void test_mockStatic_StringUtils() {
        assertTrue(StringUtils.isBlank("   "));
        PowerMockito.when(StringUtils.isBlank(any())).thenReturn(false);
        assertFalse(StringUtils.isBlank("   "));
    }

    @Test
    public void test_mockStatic_RandomStringUtils() {
        assertNull(RandomStringUtils.randomAlphabetic(10));
        PowerMockito.when(RandomStringUtils.randomAlphabetic(anyInt())).thenReturn("Hello World!");
        assertEquals("Hello World!", RandomStringUtils.randomAlphabetic(10));
    }
}

註：

  • 我地用既係 SpringRunner 而唔係 PowerMockRunner。
    • 正因為咁，所以我地要加 @Rule public PowerMockRule powerMockRule = new PowerMockRule();
      • 呢個 property 一定要係 public。
      • 呢個 property 要先 initialize，例如 new PowerMockRule()。
  • @PowerMockIgnore 有可能需要用到。
  • 我地要 mock static 既 class 係 Apache Commons Lang3 既 StringUtils 以及 RandomStringUtils。
  • PowerMock 使用上其實同 Mockito 係非常相似。
    • PowerMockito.mockStatic(class) 默認會令 mock object 既所有 methods 都冇任何 implementation，即係如果有 return type，佢就會返回 null 或者 false 之類既默認 value，除非我地用 when-then 既寫法賦予被測試既 method 一啲新既 behavior。
    • PowerMockito.spy(class) 係當我地需要 mock object 默認用返本身既 implementation，除非我地用 when-then 既寫法賦予被測試既 method 一啲新既 behavior。
