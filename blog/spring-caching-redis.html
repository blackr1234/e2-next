1 Redis 簡介

一般既 web applications 都會將最重要既資料（例如客戶資料、交易數據）放喺一個或者多個（如果係 microservice architecture）RDBMS 既 databases 裡面，而呢啲 databases 既資料都可以被視作為 single source of truth。不過，資料可以分好多種，除左最重要、改既話一定要直接改 database 既紀錄果啲，仲有一啲比如係用戶網頁登入既 session states、多國語言既翻譯文本，或者係一啲基本上都唔會點改變既業務配置或者程序配置。為左減少 database 既存取、加快查詢速度，我地可以用唔同既 caching library，例如 Redis、Memcached、Ehcache、Caffeine。

關於 Redis：

  • 係一個 in-memory database。
    • 佢既 data 都可以 save 落 disk（例如 Redis server shutdown 果陣，或者用 Redis command）。
  • 資料可以同時畀好多個 Redis client 既 application instances 去存取，大家存取到既資料都係一樣既。
  • 支援 transaction，隔離狀態咁一次過執行多個 commands，但係就唔 ACID（大概只可以做到 isolation，因為唔支援 rollback）。
  • 可以建立多個 replicas 以及 sentinels 去達到 high availability。
  • 支援搭建 cluster 做 distributed cache。
  • 可以視為一個好快既以讀取為主既 key-value database。

2 安裝

2.1 Docker image

docker container run -d --rm -p 6379:6379 -v "C:/docker-data/redis:/data" --name redis redis:latest redis-server --save 10 1 --requirepass redispw

Redis Docker image 裡面已經包含 Redis CLI，所以我地只需要接連入去個 container 既 CLI 就可以用到。

連接 Redis CLI：

  • 方法一
    1. 去 Redis Docker container 頁面既右上角果度按 CLI 圖示
    2. 彈出黎既 CLI 視窗已經連接左 Redis Docker container 既 CLI，我地可以執行： 
      redis-cli -a redispw
  • 方法二
    1. 喺 Windows Command Prompt 或者 macOS Terminal 執行： 
      docker container exec -it <redis container name/ID> redis-cli -a redispw

2.2 Windows（免安裝）

  1. 去 tporadowski/redis GitHub 下載 ZIP 版既 Redis server。
  2. 解壓縮個 ZIP 檔。
  3. 執行 command：redis-server.exe redis.windows.conf

2.3 macOS（Homebrew）

  1. 執行 command：brew install redis
  2. 執行 command：redis-server

3.3 寫 Java code

3.3.1 Entity

PersonEntity.java：

@Data
@Accessors(chain = true)
@FieldDefaults(level = PRIVATE)
@Entity
@Table(name = "person", uniqueConstraints = {
    @UniqueConstraint(name = "UQ_person_1", columnNames = { "email" })
})
public class PersonEntity {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    @Column(name = "id")
    Long id;

    @Column(nullable = false)
    String email;

    String name;
}

解釋：

  • id
    • Database 層面 auto-increment
  • email
    • Database 層面 guarantee 一定唔會 null
    • Database 層面 guarantee 一定唔會重複

3.3.2 Repository

PersonRepo.java：

import org.springframework.cache.annotation.Cacheable;

@CacheConfig(cacheManager = "personCache")
@Repository
public interface PersonRepo extends JpaRepository<PersonEntity, Long> {

    @CachePut(value = "personByEmail", key = "#entity.email", unless = "#result.id == null")
    @Override <S extends PersonEntity> S save(S entity);

    @Cacheable(value = "personByEmail", key = "#email", unless = "#result == null")
    Optional<PersonEntity> findOneByEmail(String email);

    @CacheEvict(value = "personByEmail", key = "#email")
    void deleteByEmail(String email);
}

解釋：

  • @CachePut
    • 用 return value 覆蓋或者新增 cache 紀錄。
    • value 以及 key 會組合成 cache 既 key 名，例如 personByEmail::michaelboyboy@gmail.com。
    • unless 用黎喺攞到真既 return value 之後，根據自定義既邏輯去選擇性咁去做定唔做相關操作。
  • @Cacheable
    • 如果 cache 已經存在相關紀錄，返回 cache 既紀錄；否則，call 真 method（訪問 database），然後將 return value 放入 cache。
    • value 以及 key 會組合成 cache 既 key 名，例如 personByEmail::michaelboyboy@gmail.com。
    • unless 用黎喺攞到真既 return value 之後，根據自定義既邏輯去選擇性咁去做定唔做相關操作。
  • @CacheEvict
    • 刪除 cache 紀錄。
    • value 以及 key 會組合成 cache 既 key 名，例如 personByEmail::michaelboyboy@gmail.com。
    • 可以用 allEntries = true 黎刪曬所有 value 下既所有 cache 紀錄（不限 key）。

3.3.3 Service

PersonService.java：

import org.springframework.beans.BeanUtils;
import org.springframework.transaction.annotation.Transactional;

@CacheConfig(cacheManager = "personCache")
@Service
@Transactional(rollbackFor = Throwable.class)
public class PersonService {

    @Autowired
    PersonRepo personRepo;

    public PersonEntity insert(PersonEntity person) {
        return personRepo.save(person);
    }

    public PersonEntity update(PersonEntity person) {
        return personRepo.findOneByEmail(person.getEmail()).map(e -> {
            BeanUtils.copyProperties(person, e, "id");
            return personRepo.save(e);
        }).orElse(null);
    }

    public PersonEntity getByEmail(String email) {
        return personRepo.findOneByEmail(email).orElse(null);
    }

    public void deleteByEmail(String email) {
        personRepo.deleteByEmail(email);
    }
}

解釋：

  • update
    1. 喺 database 或者 cache 攞出紀錄
      • Cache：透過 PersonRepo#findOneByEmail 既 @Cacheable
    2. 將 id 以外既 fields 覆蓋原有紀錄（保留 id value）
      • 利用 String 自帶既 BeanUtils#copyProperties(Object source, Object target, String... ignoreProperties)
    3. 更新 database 紀錄
      • 透過 PersonRepo#save
    4. 新增或者更新 cache 紀錄
      • 透過 PersonRepo#save 既 @CachePut

@RestController

    }

        return personService.update(person);

    @GetMapping("{email}")
    public PersonEntity get(@PathVariable("email") String email) {
        return personService.getByEmail(email);
    }

    @DeleteMapping("{email}")

        personService.deleteByEmail(email);
    }
}

3.3.5 自定義 RedisCacheManager

RedisConfig.java：

@EnableCaching
@Configuration
public class RedisConfig {

    @Bean
    public RedisCacheManager personCache(RedisConnectionFactory connectionFactory) {
        return RedisCacheManager.builder(connectionFactory)
                .withCacheConfiguration("personByEmail", personRedisCacheConfig())
                .build();
    }

    @Bean
    public RedisCacheConfiguration personRedisCacheConfig() {
        return RedisCacheConfiguration.defaultCacheConfig()
                .disableCachingNullValues()
                .entryTtl(Duration.ofMinutes(10))
                .serializeKeysWith(SerializationPair.fromSerializer(RedisSerializer.string()))
                .serializeValuesWith(SerializationPair.fromSerializer(RedisSerializer.json()));
    }
}

解釋：

  • @EnableCaching
    • 一定要加上呢個 annotation 先會開啟 caching 功能，否則所有 caching 相關既 annotations 都唔會生效。
  • 配置 redisCacheConfiguration
    • 根據 RedisCacheConfiguration#defaultCacheConfig，Redis caching 默認既 value serializer 係 JdkSerializationRedisSerializer。
      • 喺 runtime 既時候，當需要 cache 既 method 被 call 完之際，return value 就要被 cache。
      • JdkSerializationRedisSerializer 所配置既 DefaultSerializer 係會 check 要 serialize 既對象係咪 Serializable。
      • 但因為 return type 係 PersonEntity、List<PersonEntity 或者 Optional<PersonEntity>，實際對象 PersonEntity 唔係一個 Serializable。
      • 咁就會導致 exception DefaultSerializer requires a Serializable payload but received an object of type [code.entity.PersonEntity]。
        • 有 2 個方法解決
          • 我地可以將 PersonEntity 改成 implement Serializable。
          • 如果我地想將 cache value 儲成 JSON 格式，就要將 value serializer 改成用 Jackson Databind 既 GenericJackson2JsonRedisSerializer（相當於 RedisSerializer.json()）。
    • disableCachingNullValues
      • Return value 係 null 既話都會出 exception，除非有 unless。
    • entryTtl
      • Cache 紀錄既存活時間（time-to-live）。
