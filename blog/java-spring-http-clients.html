Java Spring Boot／Cloud HTTP clients

1 Java Spring 提供既 HTTP client APIs
微服務（又或者 monolith applications）可以透過 HTTP clients 以 HTTP REST calls 既形式連接其他服務，包括獲取、上載、更新、刪除資料。
以下係 Java Spring Boot 或者 Spring Cloud 項目可以用到既 HTTP client APIs：
  1. Java HttpClient
  2. Spring RestTemplate
  3. Spring Webflux WebClient
  4. Spring 6.1 RestClient（新）
  5. Spring Cloud OpenFeign

2 動手寫

2.1 Java HttpClient
以下係 Java 本身提供既 HttpClient API。
構建 client object：
final HttpClient client = HttpClient.newBuilder()
	.version(Version.HTTP_1_1)
	.connectTimeout(Duration.ofSeconds(10))
	.build();
發送 request：
final String requestBody = "{ \"value\": \"123\" }";

final HttpRequest request = HttpRequest.newBuilder()
	.version(Version.HTTP_1_1)
	.uri(new URI("http://localhost:8081/api/order/123?param1=value&param2=value"))
	.POST(BodyPublishers.ofString(requestBody, StandardCharsets.UTF_8))
	.header("Authorization", "Basic " + Base64.getEncoder().encodeToString(("username" + ":" + "password").getBytes()))
	.header("Content-Type", "application/json")
	.header("header1", "header1-value")
	.header("header2", "header2-value")
	.build();

final HttpResponse<String> response = client.send(request, BodyHandlers.ofString(StandardCharsets.UTF_8));
2xx、3xx、4xx 或 5xx responses：
final String responseBody = response.body();
final int responseStatusCode = response.statusCode();
final boolean isResponse2xx = responseStatusCode >= 200 && responseStatusCode < 300;
final boolean isResponse3xx = responseStatusCode >= 300 && responseStatusCode < 400;
final boolean isResponse4xx = responseStatusCode >= 400 && responseStatusCode < 500;
final boolean isResponse5xx = responseStatusCode >= 500 && responseStatusCode < 600;
final Map<String, List<String>> responseHeaders = response.headers().map();

2.2 Spring RestTemplate
以下係 Spring 一直以黎都有既 RestTemplate API。自從 Spring 5，呢個 class 已經係喺 maintenance mode，Spring 建議我地轉用 WebClient。
構建 client object：
final RestTemplate client = new RestTemplateBuilder()
	.setConnectTimeout(Duration.ofSeconds(10))
	.basicAuthentication("username", "password")
	.defaultHeader("header1", "header1-value")
	.defaultHeader("header2", "header2-value")
	.build();

// 如果加入左 jackson-dataformat-xml
// 令 RestTemplate 唔好喺 Accept request header 度加入 application/xml 或相關既 values
client.getMessageConverters().removeIf(e -> e.getClass()==MappingJackson2XmlHttpMessageConverter.class);
發送 request：
final String requestBody = "{ \"value\": \"123\" }";
final HttpHeaders requestHeaders = new HttpHeaders();
requestHeaders.set(HttpHeaders.CONTENT_TYPE, MediaType.APPLICATION_JSON_VALUE);
requestHeaders.set("header1", "header1-value");
requestHeaders.set("header2", "header2-value");

try {
	final ResponseEntity<String> response = client.exchange(
		"http://localhost:8081/api/order/{orderId}?param1={param1}&param2={param2}",
		HttpMethod.POST,
		new HttpEntity<>(requestBody, requestHeaders),
		String.class,
		"123",
		"param1-value",
		"param2-value");
} catch (RestClientResponseException e) {
	// 4xx 或 5xx responses
}
2xx 或 3xx responses：
final String responseBody = response.getBody();
final int responseStatusCode = response.getStatusCode().value();
final boolean isResponse2xx = response.getStatusCode().is2xxSuccessful();
final boolean isResponse3xx = response.getStatusCode().is3xxRedirection();
final Map<String, List<String>> responseHeaders = response.getHeaders();
4xx 或 5xx responses：
} catch (RestClientResponseException e) {
	final String responseBody = e.getResponseBodyAsString();
	final int responseStatusCode = e.getStatusCode().value();
	final boolean isResponse4xx = e.getStatusCode().is4xxClientError();
	final boolean isResponse5xx = e.getStatusCode().is5xxServerError();
	final Map<String, List<String>> responseHeaders = e.getResponseHeaders();
}
注意：
  • 某啲 classpath libraries（Maven dependencies）會影響到 Spring RestTemplate 幫我地 send HTTP calls 既時候加入既 Accept request header。
    • 創建 Spring RestTemplate 既時候，佢會因為喺 classpath 存在既 classes 而去推斷 Accept request header。
    • 例如 classpath 度有 jackson-dataformat-xml 既話，Spring RestTemplate 就會用左 Accept: text/plain, application/xml, text/xml, application/json, application/*+xml, application/*+json, */* 既 request header 去 send HTTP calls。
    • 如果呢啲 libraries 係必需，咁我地可以透過刪除 Spring RestTemplate object 裡面既 message converters 去改變佢既 behavior。
詳情可以睇返呢篇：Spring JSON 變 XML response 問題。

2.3 Spring Webflux WebClient
以下係黎自 Spring Webflux 既 WebClient API。佢支援 synchronous（blocking）以及 asynchronous（non-blocking）既 HTTP calls。
Spring Boot servlet WebMVC 既項目只要加入 Spring Webflux 既 library 就用得到呢個 client。
構建 client object：
final HttpClient httpClient = HttpClient.newBuilder()
	.version(Version.HTTP_1_1)
	.connectTimeout(Duration.ofSeconds(10))
	.build();

final WebClient client = WebClient.builder()
	.clientConnector(new JdkClientHttpConnector(httpClient))
	.defaultHeader(HttpHeaders.CONTENT_TYPE, MediaType.APPLICATION_JSON_VALUE)
	.defaultHeader("header1", "header1-value")
	.defaultHeader("header2", "header2-value")
	.build();
發送 request：
final String requestBody = "{ \"value\": \"123\" }";

try {
	final ResponseEntity<String> response = client.post()
		.uri("http://localhost:8081/api/order/{orderId}?param1={param1}&param2={param2}", "123", "param1-value", "param2-value")
		.headers(headers -> headers.setBasicAuth("username", "password"))
		.header(HttpHeaders.CONTENT_TYPE, MediaType.APPLICATION_JSON_VALUE)
		.header("header1", "header1-value")
		.header("header2", "header2-value")
		.bodyValue(requestBody)
		.retrieve()
		.toEntity(String.class)
		.block();
} catch (WebClientResponseException e) {
	// 4xx 或 5xx responses
}
2xx 或 3xx responses：
final String responseBody = response.getBody();
final int responseStatusCode = response.getStatusCode().value();
final boolean isResponse2xx = response.getStatusCode().is2xxSuccessful();
final boolean isResponse3xx = response.getStatusCode().is3xxRedirection();
final Map<String, List<String>> responseHeaders = response.getHeaders();
4xx 或 5xx responses：
} catch (WebClientResponseException e) {
	final String responseBody = e.getResponseBodyAsString();
	final int responseStatusCode = e.getStatusCode().value();
	final boolean isResponse4xx = e.getStatusCode().is4xxClientError();
	final boolean isResponse5xx = e.getStatusCode().is5xxServerError();
	final Map<String, List<String>> responseHeaders = e.getHeaders();
}

2.4 Spring 6.1 RestClient
以下係 Spring 6.1 新加入既 RestClient API。佢支援 synchronous（blocking）既 HTTP calls。
用呢個 client 既話，就唔需要再喺 Spring Boot servlet WebMVC 既項目加入 Spring Webflux 既 library。
構建 client object：
final RestClient client = RestClient.builder()
	.defaultHeader(HttpHeaders.CONTENT_TYPE, MediaType.APPLICATION_JSON_VALUE)
	.defaultHeader("header1", "header1-value")
	.defaultHeader("header2", "header2-value")
	.build();
發送 request：
final String requestBody = "{ \"value\": \"123\" }";

try {
	final ResponseEntity<String> response = client.post()
		.uri("http://localhost:8081/api/order/{orderId}?param1={param1}&param2={param2}", "123", "param1-value", "param2-value")
		.headers(headers -> headers.setBasicAuth("username", "password"))
		.header(HttpHeaders.CONTENT_TYPE, MediaType.APPLICATION_JSON_VALUE)
		.header("header1", "header1-value")
		.header("header2", "header2-value")
		.body(requestBody)
		.retrieve()
		.toEntity(String.class);
} catch (RestClientResponseException e) {
	// 4xx 或 5xx responses
}
2xx 或 3xx responses：

2.5 Spring Cloud OpenFeign
Spring Cloud OpenFeign 將開源既 Feign library（本來係 Netflix，但之後開源成為 OpenFeign）整合到 Spring Cloud framework 裡面。
用 Spring Cloud OpenFeign 既好處係佢係 declarative，基本上將需要用到 upstream apps 既 declarative request mappings 搬入黎我地既 downstream app project 就得。
構建 client object：
@FeignClient(
	name = "fooClient",
	configuration = FooFeignConfig.class
)
public interface FooFeignClient {

	// 注意：
	// consumes ➜ Content-Type request header
	// produces ➜ Accept request header

	@PostMapping(path = "/api/order/{orderId}", consumes = MediaType.APPLICATION_JSON_VALUE)
	ResponseEntity<String> api(@PathVariable("orderId") String orderId,
								@RequestParam(name = "param1", required = false) String param1,
								@RequestParam(name = "param2", required = false) String param2,
								@RequestBody String body,
								@RequestHeader("header1") String header1,
								@RequestHeader("header2") String header2);
}
Basic auth Spring configuration：
// 如果加 @Configuration，呢個 config 就會係所有 Feign clients 既默認 config
public class FooFeignConfig {

	@Bean
	public BasicAuthRequestInterceptor fooBasicAuthInterceptor() {
		return new BasicAuthRequestInterceptor("username", "password");
	}
}
Application 配置：
spring.cloud.openfeign.client.config:
  default:
    connectTimeout: 10000
    loggerLevel: FULL
  fooClient:
    connectTimeout: 10000
    loggerLevel: FULL
    url: http://localhost:8081
    defaultRequestHeaders:
      header1: header1-value
      header2: header2-value

logging:
  level:
    my.package.FooFeignClient: DEBUG
發送 request：
@Autowired
FooFeignClient client;

public void sampleCall() {
	final String requestBody = "{ \"value\": \"123\" }";

	try {
		final ResponseEntity<String> response =
			client.api("123", "param1-value", "param2-value", requestBody, "header1-value", "header2-value");
	} catch (FeignException e) {
		// 3xx、4xx 或 5xx responses
	}
}
2xx responses：
final String responseBody = response.getBody();
final int responseStatusCode = response.getStatusCode().value();
final boolean isResponse2xx = response.getStatusCode().is2xxSuccessful();
final Map<String, List<String>> responseHeaders = response.getHeaders();
3xx、4xx 或 5xx responses：
} catch (FeignException e) {
	final String responseBody = e.contentUTF8();
	final int responseStatusCode = e.status();
	final boolean isResponse3xx = e.status() >= 300 && e.status() < 400;
	final boolean isResponse4xx = e.status() >= 400 && e.status() < 500;
	final boolean isResponse5xx = e.status() >= 500 && e.status() < 600;
	final Map<String, Collection<String>> responseHeaders = e.responseHeaders();
}
注意：
  • 需要喺某個 Spring config class 度加上 @EnableFeignClients，否則唔會生成 Feign client objects。
  • Feign clients 裡面既 request mapping methods 既 API contract 應該係要 match 返 target upstream app 既 request mapping methods。
    • consumes ➜ Content-Type
    • produces ➜ Accept
    • ❌ 唔好以為 produces 指既係呢個 app 所產生既 request body 既 Content-Type，咁樣係錯既。
  • 如果唔畀 consumes attribute，佢會根據 method signature 自動推斷 Content-Type request header 既 value。
  • 就算 Feign client method 既 return type 係 ResponseEntity，當 response status code 唔係 2xx 既時候，佢會照 throw exception。
    • ❌ 唔好諗住 return ResponseEntity 就可以拎到 response status code 然後 apply 自己既 logic，呢個係做唔到既。
  • ⚠️ 某啲 Spring Cloud OpenFeign 4.x 版本有個 bug，當我地喺 @FeignClient 定義左 path attribute（以免喺 request mapping annotations 度重複 common path prefix），而 target upstream app 既 URL 係喺 application 配置檔度定義，佢就會出 exception target values must be absolute.，呢個 bug 應該會喺後期版本修復。
